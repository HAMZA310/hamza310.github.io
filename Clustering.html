Kmeans Clustering¶

So far, we dealt with datasets in which we're also given a label for each datapoint (supervised setting). Now, we'll deal with datasets in which we don't know the labels (unsupervised setting). In this tutorial, given a training set, we want to group the data into a few cohesive clusters- called clustering.

Algorithm:

  • Initilaize k clusters randomly (we can pick any k training examples).

  • Repeat Until Convergence:

    1. Assign each training example to a cluster based on how close it is to a cluster.

    2. Compute mean of all the points assigned to cluster i- that mean is your new guess for cluster i.

K-means¶

Let's consider a randmom synthetic datset.

Out[5]:

Even if you werent to look at the code that generated these points, it should be fairly apparent that there are two different clusters of points here. The goal of the k-means algorithm is to find a set of k centers (that is, points (i)Rn) such that every data point is close to at least one center. By doing so, we can also associate each point with its closest center, and use this as an indication of which cluster it belong to.

Using these closest indexes we can cluster the closest examples.

In the code below, keeping track of datapoints already assigned to a cluster, we simply put the closest datapoints for each example in one cluster (a simple approach).

clusters = defaultdict(list)
already_assigned_datapoints = set()
cluster_id = 0
for idx, closest_datapoints in enumerate(closes_points_indexes):
    "Add these points to set- to keep track"
    # first add this element
    if idx not in already_assigned_datapoints:
        already_assigned_datapoints.add(idx)
        clusters[cluster_id].append(idx)
    # now add its closed neighbours
    for _datapoint in closest_datapoints:
        if _datapoint not in already_assigned_datapoints:
            already_assigned_datapoints.add(_datapoint)
            clusters[cluster_id].append(_datapoint)
    cluster_id = cluster_id + 1
